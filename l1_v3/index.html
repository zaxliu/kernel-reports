Anomalies & Caveats

  • P32 = 3458.6×: L1 verifier 异常 (kernel ms 测得过短),非 kernel 退步;mean 应剔除该值。
  • P35 OOM 永久 failed: 7.3 GiB output tensor + 31 GiB 5090 显存上限,结构性 OOM。
  • L1 verify ako4all-only: 直接 paired 比较只能用 71 个 ako4all tag;program_md/kb 的"L1 vs v3"对比降级为 newly-real 框架。
  • claim vs verified gap: v3 claim mean=9.47× / median=1.50× vs verified mean=1.79× / median=1.10×。差距来自 fp16 atol=1e-2 高估。
  • KERNEL_SOURCE 字段覆盖: 仅 75/300 (5/26 后跑) 有该字段,225/300 老 meta 字段缺失。

本轮 fix 影响评估

  • harness scheduler 3 bug fix (KERNEL_SOURCE / MAX_ATTEMPTS retire / reset_post_anomaly 复活) 救回若干 silently-failed job,但具体 delta 缺测量 hook,无法精确量化。
  • compute-sanitizer gate 拦 misalign — 与本报告无直接关联。
  • retest17 把 13 个 baseline_self (~1.0×) + 4 个 BMC-edge case 重跑,baseline_self 13→0;ako4all paired 仍是净 -3,说明 retest 的 speedup 增益 < 1×,核心价值是 消除假 done 而非涨速。

Limitations & Next

  • L2/L3 还没跑 v3,本报告仅 L1。
  • 退步 4 项 (P14/P91/P97/P18) 未根因深查 — 是 agent stochastic 还是 verifier 严格化?需对每个 problem 比对 L1 vs v3 的 kernel 代码 + reference_time_ms。
  • 18 newly-fake 集中在 program_kb/program_md,与 fp16 atol bias 一致,这两个框架在 KernelBench L1 上仍显著靠 fp16 走捷径。
  • token / wall_time 维度未统计,可后续从 trajectory + claude_session.log 抓。
